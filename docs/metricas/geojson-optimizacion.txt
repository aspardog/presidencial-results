PROBLEMA IDENTIFICADO
---------------------
Cuando el usuario selecciona un departamento, se descarga el archivo
completo municipios.json (4.43 MB) aunque solo necesita los municipios
de ese departamento (~138 KB promedio).

Desperdicio: 97% de los datos descargados no se usan.

================================================================================
INSTRUCCIONES DE MEDICIÓN
================================================================================

Para medir en el navegador (Chrome DevTools):

1. Abrir DevTools (F12) > Network
2. Marcar "Disable cache"
3. Recargar la página
4. Anotar tamaño total transferido
5. Hacer clic en un departamento
6. Anotar tamaño del archivo GeoJSON descargado
7. Anotar tiempo de carga (columna "Time")

Métricas clave:
- Tamaño transferido (Size column)
- Tiempo hasta renderizado (Time column)
- Time to First Byte (TTFB)

Archivos a observar:
- /api/mapas/municipios/{codigo}.json (ej: 05.json para Antioquia)


================================================================================
NOTAS TÉCNICAS
================================================================================

1. El archivo municipios.json único se mantiene para compatibilidad
   (se puede eliminar en futuras versiones).

2. La propiedad 'ganador' se mantiene en el GeoJSON porque es necesaria
   para colorear el mapa. Los datos detallados de votos están en
   departamentos/municipios.json.

3. El código de MapaElectoral.tsx ahora hace fetch a:
   /api/mapas/municipios/${departamentoSeleccionado}.json

4. Se limpia el estado cuando se cambia de departamento para evitar
   mostrar datos del departamento anterior.
