================================================================================
ESTRUCTURA DEL PROYECTO - ANALISIS ELECTORAL COLOMBIA
================================================================================

Fecha de actualizacion: 2026-06-16

Este documento describe la estructura vigente del repositorio y distingue los
componentes implementados de las carpetas reservadas para trabajo futuro.


================================================================================
1. RESUMEN
================================================================================

El proyecto tiene dos superficies principales:

1. Pipeline R en la raiz del repositorio.
   - Arquitectura Medallion: Bronze -> Silver -> Gold.
   - Procesa resultados electorales de la Registraduria.
   - Genera agregaciones nacionales, departamentales y municipales.

2. Dashboard estatico en dashboard/web/.
   - Next.js con export estatico.
   - Consume JSON generados desde data/gold.
   - Despliegue publico: https://voto-colombia-2026.vercel.app

Flujo de datos:

  data/bronze/  ->  data/silver/  ->  data/gold/  ->  dashboard/web/public/api/

3. DATA
================================================================================

data/ esta excluida de Git porque contiene archivos grandes o derivados. La
estructura se puede crear o reparar con:

  Rscript scripts/00_setup/crear_estructura.R

CAPA BRONZE
--------------------------------------------------------------------------------

Proposito:
  Conservar datos crudos y metadata de ingesta.

Rutas principales:
  data/bronze/raw/electoral/
  data/bronze/raw/geograficos/
  data/bronze/metadata/electoral/

Ruta que lee el script Bronze -> Silver:
  data/bronze/raw/electoral/registraduria_2026-06-15/

Nota:
  Si los CSV existen en otra ruta, `limpieza_datos.R` no los usa como Bronze.
  En ese caso reutiliza `data/silver/electoral/datos_master.rds` si existe.

7. DEPENDENCIAS
================================================================================

R:
  - Dependencias directas declaradas en DESCRIPTION.
  - Versiones reproducibles en renv.lock.
  - Flujo recomendado:

      Rscript --vanilla scripts/00_setup/install_packages.R
      Rscript --vanilla scripts/00_setup/verificar_dependencias.R

Node:
  - Dependencias del dashboard en dashboard/web/package.json.
  - Lockfile en dashboard/web/package-lock.json.
  - Node 20.9 o superior para Next.js 16.

Python:
  - Python 3 para scripts geograficos y verificacion GeoJSON.
  - tools/geo/requirements.txt contiene dependencias auxiliares para trabajo
    geoespacial.


================================================================================
8. GIT
================================================================================

Se versiona:
  - Codigo R, Python, TypeScript y JavaScript.
  - Documentacion.
  - Manifiestos de dependencias.
  - Configuracion del dashboard.

No se versiona:
  - data/
  - logs/
  - output/
  - renv/library/
  - node_modules/
  - .next/
  - out/
  - archivos CSV, Parquet y RDS grandes.
